# Neo4j

Neo4j provides SPARK's graph database capabilities, powering social network analysis, content recommendations, and relationship-driven discovery features. Its property graph model enables complex relationship traversals that would be impractical in relational databases.

## Graph Model

The graph consists of nodes representing users, content items, categories, and tags. Relationships capture follows, likes, shares, comments, views, and content similarity. Each relationship carries properties such as timestamp, weight, and interaction type to enable time-decayed relevance scoring.

## Key Queries

**Social Feed.** A user's feed is generated by traversing the follow graph to find content from followed creators, then expanding through like and share relationships to discover trending content within the user's extended network.

**Recommendations.** The collaborative filtering engine uses common neighbor analysis to recommend creators and content. The query identifies users with similar interaction patterns and surfaces content those users engaged with but the target user has not seen.

**Influence Scoring.** PageRank and betweenness centrality algorithms identify influential users within the network. These scores feed into the trending and ranking algorithms to amplify content from meaningful connections.

## Performance

All relationship types that are traversed together are indexed as composite indexes. Hot nodes are cached in memory using the page cache which is allocated 75% of available heap. Queries are profiled using PROFILE and optimized to avoid unnecessary expansion.
