# Technology Stack

Spark uses modern, battle-tested technologies chosen for performance, scalability, and developer experience.

## Frontend

| Technology | Use |
|-----------|-----|
| Next.js | Web application framework (SSR, ISR, edge functions) |
| React 18 | UI component library with concurrent features |
| TypeScript | Type safety across the entire frontend codebase |
| Flutter | Cross-platform mobile apps (iOS and Android) |

## Backend

| Technology | Use |
|-----------|-----|
| Go | High-performance microservices (API gateway, auth, streaming) |
| Rust | Media transcoding, real-time processing, safety-critical services |
| Python | ML inference pipelines, data processing, automation |

## Data & Messaging

| Technology | Use |
|-----------|-----|
| PostgreSQL | Primary OLTP database (user data, content metadata, transactions) |
| ClickHouse | Real-time analytics, creator dashboards, recommendation features |
| Redis | Caching, session storage, rate limiting, real-time leaderboards |
| OpenSearch | Full-text search, content discovery, log aggregation |
| Kafka | Event bus — service decoupling, stream processing, data replication |

## Infrastructure

| Technology | Use |
|-----------|-----|
| Kubernetes | Container orchestration across all environments |
| Terraform | Infrastructure-as-Code for cloud resources |
| Envoy | API gateway, service mesh, edge proxy |
| Docker | Local development and container packaging |
| GitHub Actions | CI/CD pipelines |

## Observability

| Technology | Use |
|-----------|-----|
| OpenTelemetry | Distributed tracing, metrics, and log collection |
| Prometheus | Metrics aggregation and alerting |
| Grafana | Dashboards and visualization |
| Sentry | Error tracking and performance monitoring |
